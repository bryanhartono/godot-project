# Monster Tactics — Phase 3: AI + Skirmish Mode Design Spec

**Date:** 2026-05-28
**Phase:** 3 of 6
**Goal:** Replace the hot-seat prototype with a real single-player skirmish loop: random squads, interactive deployment, a heuristic AI opponent at three difficulty levels, and a win/lose screen.

---

## 1. Architecture Principles

- **Robust over fast.** Architecture is designed to support all remaining phases without structural rewrites.
- **State machine for all game flow.** Every phase of a match (deploy, player turn, AI turn, win/lose) is an explicit state object. Adding new states in future phases means adding a new file, not modifying existing ones.
- **Pure logic classes.** `TacticsAI` and `SquadPicker` are `RefCounted` — no Node, no scene dependency. They are independently testable.
- **No globals for match data.** A `MatchConfig` resource carries setup data between scenes; no autoloads or singletons for transient match state.

---

## 2. State Machine

`match_view.gd` is refactored into a thin host. It owns:
- Board rendering (tiles, Polygon2D)
- Sprite management (`AnimatedSprite2D` per unit)
- `MatchState` instance
- UI labels and overlay layers
- A `current_state: BaseBattleState` reference

All input handling and turn logic delegates to the current state.

### BaseBattleState interface
```
# scripts/battle/states/base_battle_state.gd
class_name BaseBattleState
extends RefCounted

func enter(ctx: MatchView) -> void: pass
func exit(ctx: MatchView) -> void: pass
func handle_input(ctx: MatchView, event: InputEvent) -> void: pass
```

### States

| State | File | Responsibility |
|---|---|---|
| `DeployState` | `states/deploy_state.gd` | Player places units on back 2 rows |
| `PlayerTurnState` | `states/player_turn_state.gd` | Human select → move → attack/ability |
| `AiTurnState` | `states/ai_turn_state.gd` | Run TacticsAI, apply actions, transition |
| `WinLoseState` | `states/win_lose_state.gd` | Show result overlay, restart or menu |

### Transition table
```
DeployState      → PlayerTurnState   (all player units placed)
PlayerTurnState  → AiTurnState       (End Turn pressed or all units acted)
AiTurnState      → PlayerTurnState   (AI finished all actions)
PlayerTurnState  → WinLoseState      (winner() != -1 after any action)
AiTurnState      → WinLoseState      (winner() != -1 after any action)
```

## 3. Deployment Phase (`DeployState`)

- Both squads generated by `SquadPicker.random_squad(budget: 10)` before the state is entered.
- AI squad is placed automatically: spread across columns in rows y=0 and y=1.
- Player queue: 3 unplaced `MonsterData` items displayed in a sidebar panel.
- Valid placement tiles: rows y=5 and y=6 (bottom 2 rows), unoccupied.
- On tile tap: pop the front of the queue, spawn the unit, highlight next unit.
- "Auto-place" button: fills remaining slots randomly for fast restarts.
- On queue empty: transition to `PlayerTurnState`.

## 4. TacticsAI

**File:** `scripts/battle/tactics_ai.gd`
**Type:** `RefCounted` (no Node)

### Action value type
```gdscript
# Inline in tactics_ai.gd
class Action:
    var unit: BattleUnit
    var type: int  # MOVE, ATTACK, ABILITY, END_TURN
    var target: Vector2i
```

### Heuristic scorer
```
score(state) = Σ(ally_hp) - Σ(enemy_hp) + kill_bonus * kills_this_turn
```
`kill_bonus = 5` (killing a unit is worth 5 HP equivalent). Tunable constant.

### Difficulty levels

**Easy (depth=1) — Greedy independent:**
For each AI unit in sequence, enumerate all legal move+action combinations, pick the one with the highest immediate `score()` delta. Units act selfishly; no cross-unit coordination.

**Normal (depth=2) — Team coordination:**
Enumerate all permutations of (unit, move, action) across the full 3-unit team. Pick the sequence with the highest combined `score()` delta. Units coordinate (soften + finish).

**Hard (depth=3) — Opponent lookahead:**
Same as Normal, but after selecting the best team action sequence, simulate the player's greedy best response (depth-1 for player) and select the AI sequence that minimises that counter. One ply of opponent lookahead.

### Simulation constraint
`TacticsAI` never mutates `MatchState` directly. All evaluation uses a lightweight `MatchState.duplicate()` clone. The real board is only mutated when `AiTurnState` applies the returned `Array[Action]` one by one with a small visual delay between actions.

### Public API
```gdscript
func get_actions(state: MatchState, ai_team: int, difficulty: int) -> Array[Action]
```

## 5. SquadPicker

**File:** `scripts/battle/squad_picker.gd`
**Type:** static functions only

```gdscript
static func random_squad(budget: int) -> Array[MonsterData]
```

Algorithm: shuffle all monsters, iterate the shuffled list, greedily add each monster if its cost fits within the remaining budget. Stop when the list is exhausted or no remaining monster fits. Returns 1–3 monsters (always at least 1 since the cheapest monster costs 1).

No backtracking required at 3-unit scale with a 10-cost budget.

## 6. MatchConfig

**File:** `scripts/battle/match_config.gd`
**Type:** `Resource` (not persisted to disk)

```gdscript
class_name MatchConfig
extends Resource

var player_squad: Array[MonsterData]
var enemy_squad:  Array[MonsterData]
var difficulty:   int  # 1=Easy 2=Normal 3=Hard
```

Passed from `skirmish_setup.gd` into `match_view.gd` via `get_node()` or scene parameter (Godot 4 `SceneTree.change_scene_to_packed` with metadata). No autoloads involved.

## 7. Skirmish Setup Scene

**Scene:** `scenes/battle/skirmish_setup.tscn`
**Script:** `scripts/battle/skirmish_setup.gd`

UI elements (portrait layout):
1. **Title label** — "Skirmish"
2. **Squad preview panel** — shows 3 randomly drawn player monsters (sprite icon + name). "Reroll" button draws a new squad.
3. **Difficulty selector** — three toggle buttons: Easy / Normal / Hard. Default: Normal.
4. **Start button** — builds `MatchConfig`, switches to `match_view.tscn`.

`project.godot` main scene becomes `skirmish_setup.tscn`.

---

## 8. Win/Lose State (`WinLoseState`)

Triggered when `MatchState.winner()` != -1.

Renders a semi-transparent overlay on top of the frozen board:
- **Result banner** — "Victory!" or "Defeat" 
- **"Play Again" button** — generates fresh squads (same difficulty), reloads match_view
- **"Menu" button** — returns to skirmish_setup.tscn

Board remains visible underneath (units frozen). No stats, no rewards — those are Phase 4.

**Phase 5 hook:** The interstitial ad slot fires here, before the overlay appears. `WinLoseState.enter()` will call `AdsManager.show_interstitial(on_closed_callback)` in Phase 5; for now it's a no-op comment placeholder.
